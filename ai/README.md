# AI README

Este directorio contiene el contexto y las reglas que debe leer cualquier IA antes de trabajar sobre este proyecto.

## Orden de lectura

Antes de analizar, modificar o generar código, leer en este orden:

1. `ai/context.md`
2. `ai/models.md`
3. `ai/arquitectura.md`
4. `ai/skills.md`

## Propósito

Este archivo funciona como punto de entrada para la IA.

Su objetivo es indicar qué documentación debe revisar antes de responder o modificar el proyecto.

## Reglas iniciales

- No generar código si no fue solicitado explícitamente.
- No modificar estructura sin revisar primero la documentación de `ai/`.
- No asumir funcionalidades fuera del alcance definido.
- Mantener respuestas y cambios alineados con el contexto del proyecto.
- Priorizar cambios pequeños, claros y controlados.

## Criterio general

Si una tarea afecta arquitectura, modelos, estructura de carpetas o reglas de trabajo, revisar primero el archivo correspondiente antes de proponer cambios.
